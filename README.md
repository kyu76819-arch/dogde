# Discord Ticket Bot Server

이 프로젝트는 기존 티켓 시스템의 디스코드 연동 기능을 서버 사이드로 분리한 Node.js 기반의 디스코드 봇입니다. 클라우드타입(Cloudtype) 환경에 최적화되어 있으며, GitHub에 안전하게 공개할 수 있도록 민감 정보는 환경 변수로 관리합니다.

## 주요 기능

- **티켓 생성 알림**: 새로운 티켓이 생성되면 관리자에게 DM으로 알림을 전송합니다.
- **실시간 채팅 연동**: 웹 애플리케이션과 디스코드 DM 간의 메시지를 실시간으로 전달합니다.
- **티켓 상태 관리**: 디스코드 내 버튼 상호작용을 통해 티켓을 닫거나 상태를 변경할 수 있습니다.
- **메시지 추적 및 삭제**: 티켓이 닫히거나 삭제되면 관련 DM 메시지를 자동으로 삭제하여 보안을 유지합니다.

## 설치 및 배포 가이드

### 1. 사전 준비

- [Discord Developer Portal](https://discord.com/developers/applications)에서 봇을 생성하고 토큰을 준비합니다.
- Firebase 프로젝트를 생성하고, Realtime Database를 활성화합니다.
- Firebase 서비스 계정 키(JSON 파일)를 생성하여 내용을 복사해 둡니다.

### 2. 환경 변수 설정

클라우드타입 배포 시 다음 환경 변수를 설정해야 합니다:

| 변수명 | 설명 | 비고 |
| :--- | :--- | :--- |
| `DISCORD_BOT_TOKEN` | 디스코드 봇 토큰 | 필수 |
| `FIREBASE_SERVICE_ACCOUNT_KEY` | Firebase 서비스 계정 키 (JSON 문자열) | 필수 |
| `FIREBASE_DATABASE_URL` | Firebase Realtime Database URL | 필수 |
| `PORT` | 서버 포트 (기본값: 3000) | 선택 |

### 3. 클라우드타입 배포

1. GitHub 저장소에 이 프로젝트를 업로드합니다.
2. 클라우드타입에서 새로운 서비스를 생성하고 해당 GitHub 저장소를 연결합니다.
3. 배포 설정에서 위의 환경 변수들을 입력합니다.
4. 배포가 완료되면 제공되는 URL을 확인합니다.

### 4. 웹 애플리케이션 연동

웹 애플리케이션에서 디스코드 이벤트 발생 시 클라우드타입에 배포된 서버의 `/webhook/ticket` 엔드포인트로 POST 요청을 보내도록 설정해야 합니다.
